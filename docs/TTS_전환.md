# [아키텍처 검토 보고서] AI 에이전트 오디오 파이프라인 기술 전환

본 문서는 로컬 하드웨어 인프라 및 사용자 경험(UX) 향상을 위해 AI 에이전트 프로젝트의 오디오 파이프라인(TTS 및 대화 메커니즘) 설계 변경 사항을 검토하고, 향후 개발 방향을 정의하기 위한 개발 가이드라인이다.

---

## 1. 프로젝트 환경 및 가용 리소스
*   **호스트 환경:** 로컬 데스크톱 (상시 가동)
*   **GPU 스펙:** AMD Radeon RX 6600 XT (VRAM 8GB) / ROCm 가속 환경 구축 필요
*   **LLM 운용 전략:** 로컬 자원(VRAM) 보호를 위해 **외부 LLM API 사용** 및 **텍스트 스트리밍(Streaming)** 연동 확정

---

## 2. TTS 기술 스택 전환 (Pivot)

### 기존 안 (Supertonic) 유지 불가 사유
*   **한계:** 10종의 고정 프리셋만 지원하여 개발자가 원하는 '특정 미디어/게임 캐릭터' 고유의 음색 및 감정선 표현 불가능.
*   **기능 부재:** 일본어 레퍼런스 음성 샘플을 기반으로 한국어를 유창하게 발화시키는 교차 언어(Cross-lingual) 기능 구현 불가.

### 신규 안 (Zero-shot Voice Cloning) 채택
단 3~10초의 캐릭터 오디오 샘플만으로 고유 음색을 복제하는 **오디오 파운데이션 모델** 체제로 전환한다. LLM API 스트리밍 도입으로 약 6GB의 VRAM이 확보되어 로컬 GPU 가속 연산 마진이 충분하다.

| 후보 모델 | 아키텍처 유형 | 주요 특징 |
| :--- | :--- | :--- |
| **GPT-SoVITS (V2)** | 자기회귀 (Autoregressive) | 캐릭터 음색/호흡 복제율 최상, 일-한 교차 언어 성능 검증됨 |
| **F5-TTS** | 플로우 매칭 (Flow Matching) | 최신 디퓨전 계열, 추론 속도가 빠르고 억양이 자연스러움 |
| **CosyVoice2** | 플로우 매칭 계열 | 다국어 제로샷 능력 우수, 감정 제어 디테일 강점 |

## 3. 오디오 스트리밍 및 지연 시간(Latency) 최적화

LLM의 텍스트 스트리밍 출력을 문장(또는 구절) 단위의 비동기 큐(Queue)로 분할하여 TTS 엔진에 순차 투입한다.

*   **목표 지연(TTFA):** 첫 문장 생성 및 음성 합성까지 **0.5초 내외** 달성 (체감상 실시간).
*   **버퍼링 메커니즘:** 첫 번째 문장의 음성이 스피커로 재생되는 동안, GPU는 백그라운드에서 두 번째, 세 번째 문장의 음성을 미리 합성하여 대기열에 적재함으로써 대화의 연속성을 보장한다.

---

## 4. 실시간 대화 UX: 바지인(Barge-in, 끼어들기) 아키텍처

턴제(Turn-based) 대화 방식을 탈피하여, 에이전트가 말하는 도중에도 사용자가 말을 끊고 개입할 수 있는 인간형 대화 메커니즘을 구현한다.

### A. 상시 VAD (음성 활성 감지) 구성
*   마이크 입력을 상시 개방하되, CPU 부하가 거의 없는 경량 AI 모델인 **Silero VAD**를 전면에 배치하여 0.1초 단위로 사람의 유효 발화를 감지한다.

### B. 킬 스위치(Kill Switch) 비동기 시퀀스
사용자의 끼어들기 감지 즉시 다음 오디오 파이프라인 중단 프로세스를 `Asyncio` 이벤트 루프로 동시 실행한다.
1.  **스피커:** 현재 재생 중인 오디오 즉시 중단 (`Audio.stop()`)
2.  **TTS 엔진:** 큐(Queue)에 쌓여 있던 다음 문장들의 음성 합성 작업 강제 취소 (`Queue.clear()`)
3.  **LLM 연동:** 진행 중이던 LLM API 스트리밍 HTTP/WebSocket 연결 강제 종료
4.  **STT 전환:** 마이크 입력을 Whisper(STT) 엔진으로 넘겨 신규 발화 컨텍스트 해석 시작

### C. 에코 캔슬링(AEC) 대책
*   에이전트 자신의 출력이 마이크로 재유입되어 끼어들기가 오작동하는 현상을 방지하기 위해, WebRTC AEC 라이브러리 도입 또는 하드웨어적 차단(이어셋 사용 권장) 환경을 전제한다.

---

## 5. 에이전트 행동 지침 (Action Items)
1.  **인프라 환경 확인:** AMD RX 6600 XT 환경에서 `PyTorch-ROCm` 빌드를 활용해 GPT-SoVITS V2 및 F5-TTS의 가속 인퍼런스 환경을 구성할 수 있는 가이드를 준비할 것.
2.  **데이터 전처리 로드맵:** 캐릭터 대사 파일에서 BGM 및 효과음을 제거하고 3~10초의 무음 구간이 정제된 고품질 보컬 레퍼런스를 추출하는 전처리 파이프라인(Vocal Remover 등)을 설계할 것.
3.  **코드 프레임워크 설계:** 파이썬 `asyncio`를 기반으로 `LLM 스트리밍 분할 ➡️ TTS 합성 큐 ➡️ Silero VAD 모니터링 ➡️ 오디오 인터럽트 제어`가 유기적으로 연결되는 백엔드 아키텍처 초안을 작성할 것.
